Why did I change packaged Chrome app version to hosted one?

Wow! Such a big red title! Later I'll put this entry somewhere else with less aggressive style, but this is a quite important now.

When I started to work on Dockery the plan was to learn Javascript and Angular a bit via a demo project. As I added more and more function, I started to use regularly during my daily work, so I decided to share the app with others for free. One day my colleague Jens asked me why don't try to release Dockery as Chrom app? I scratched my head, then fired up Google to search for some tutorials. Next day I released the packaged app into the Chrome shop and checked the daily statistics of the installs :) Turning an Angular app to Chrome app is relatively easy with some extra spice, but from the beginning I missed some features from the app: back button, opening URL by click, etc. I went through several tutorials, mailing archives and blog entries, but there were no answer to my questions, how to make equally usable packaged app for Chrome like the hosted version.

Finally I decided to put Dockery on Github pages first, then change the app from packaged to hosted one. I hope it remains usable for everyone, but please create an issue ticket on Github if you feel the necessity of the old, packaged app, then I will try to do something.
